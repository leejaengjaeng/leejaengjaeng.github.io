1. Hugo 설치

https://gohugo.io/getting-started/installing/

2. 프로젝트(사이트) 생성

hugo new site {이름}

프로젝트 구조

https://gohugo.io/getting-started/directory-structure/

  1. archetypes

    • hugo new 커맨드를 통해 생성될 content file에 들어갈 front matter 등을 정의해 두는 디렉터리
    • front matter란 content file에 정의된 --- 부분
    • draft 필드는 프로젝트 빌드시 포함시킬지 여부

  2. content

    • 작성할 글이 들어가는 디렉터리
    • content 내부의 최상의 디렉터리는 section으로 취급

  3. data

    • 설정 파일이 들어가는 디렉터리
    • YAML, JSON, TOML 같은 포맷을 사용

  4. layouts

    • .html 파일의 템플릿을 저장하는 디렉터리

  5. public

    • hugo로 빌드한 결과물이 들어가는 디렉터리

3. 테마 다운받기

  • 적당한 테마를 다운받아서 themes 폴더 하위에 clone
  • config.toml에 theme = "테마 이름"을 추가해서 등록

4. 글 쓰기

hugo new {이름}

  • 기본적으로 content 하위로 생성
  • front matter만 복사한다면 복사 & 붙여넣기로 생성해도 동일

5. local에 띄워보기

hugo server -D

  • 디폴트 포트는 1313으로 localhost:1313에 접근해서 확인가능
  • -D는 draft 문서 포함하는 옵션

6. 빌드 및 배포

hugo

  • 빌드가 완료되면 public 디렉토리에 static 웹 서비스가 생성
  • public 디렉터리를 git 디렉터리로 만들고 pages repo에 push 하면 github pages 블로그 완성
